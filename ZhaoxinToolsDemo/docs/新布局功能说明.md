# Demo 项目新布局功能说明

## 概述

Demo 项目已重构为统一的三栏布局：
- **顶部导航栏**：快速切换三大功能（智能文档抽取、智能文档比对、智能合同合成）
- **左侧演示文档区**：预置演示文档，一键加载使用
- **右侧功能区**：主要功能操作界面

## 新增功能

### 1. 统一的顶部导航

- **位置**：页面顶部固定
- **高度**：60px
- **功能**：
  - 显示 Logo 和应用名称
  - 提供三个主要功能的快速切换按钮
  - 当前页面高亮显示

### 2. 左侧演示文档区

- **位置**：页面左侧
- **宽度**：320px（1440px屏幕）/ 280px（1280px屏幕）
- **功能**：
  - 显示当前功能相关的演示文档列表
  - 点击文档卡片自动加载文档
  - 自动选择关联的模板（如果配置了）
  - 支持滚动查看所有文档

### 3. 响应式布局

- **最大宽度**：1920px（超宽屏自动居中）
- **适配尺寸**：
  - 1920px+：标准布局
  - 1440px：左侧栏 320px
  - 1280px：左侧栏 280px，字体略小
  - 768px以下：左侧栏 260px，导航栏简化

## 使用指南

### 准备演示文档

1. 将 PDF 文档放入对应目录：
   - 抽取：`backend/src/main/resources/demo/extract/`
   - 比对：`backend/src/main/resources/demo/compare/`

2. 确保文件名与代码中配置的一致：
   - 采购合同示例.pdf
   - 服务协议示例.pdf
   - 合同-原始版.pdf
   - 合同-修订版.pdf

3. 重启后端服务

### 配置演示文档

编辑 `DemoResourceController.java`：

```java
// 添加新的演示文档
documents.add(new DemoDocument(
    "doc_id",              // 唯一ID
    "文档名称.pdf",         // 显示名称
    "demo/extract/文档名称.pdf",  // 文件路径
    "extract",             // 分类：extract/compare/compose
    "模板名称或ID",         // 关联的模板（可选）
    "文档描述"             // 描述信息
));
```

### 演示流程

#### 智能文档抽取
1. 访问 `/extract-main`
2. 左侧选择演示文档
3. 系统自动加载文档并选择模板
4. 点击"开始提取"

#### 智能文档比对
1. 访问 `/compare`
2. 左侧依次选择两份文档（原始版和修订版）
3. 系统自动加载文档到对应位置
4. 点击"开始比对"

#### 智能合同合成
1. 访问 `/compose-main`
2. 左侧选择演示模板
3. 系统自动选择模板并跳转到填写数据步骤
4. 填写数据后生成合同

## API 接口

### 获取演示文档列表
```http
GET /api/demo/documents
```

**响应示例**：
```json
{
  "code": 200,
  "data": [
    {
      "id": "extract_demo_1",
      "name": "采购合同示例.pdf",
      "filePath": "demo/extract/采购合同示例.pdf",
      "category": "extract",
      "templateId": "采购合同模板",
      "description": "一份标准的采购合同"
    }
  ]
}
```

### 下载演示文档
```http
GET /api/demo/documents/download?path={filePath}
```

## 注意事项

1. **演示文档必须是 PDF 格式**
2. **文件大小建议不超过 10MB**
3. **文件路径必须在 resources/demo 目录下**
4. **抽取文档需要绑定已存在的模板**
5. **合成功能需要在主系统中创建模板并获取 ID**

## 已知限制

1. 当前版本演示文档列表是硬编码在后端
2. 合成功能的演示模板需要手动在主系统中创建
3. 移动端适配尚不完善（建议桌面端使用）
