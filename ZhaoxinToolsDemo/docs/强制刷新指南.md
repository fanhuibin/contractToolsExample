# 🔄 强制刷新指南

## 浏览器缓存清除方法

如果修改后的样式没有生效，请按照以下步骤**强制刷新**浏览器：

### 方法1：键盘快捷键（推荐）⌨️

#### Windows/Linux
```
Ctrl + Shift + R
或
Ctrl + F5
```

#### Mac
```
Cmd + Shift + R
或
Cmd + Option + R
```

### 方法2：开发者工具刷新 🛠️

1. 按 `F12` 打开开发者工具
2. **右键点击**浏览器刷新按钮 🔄
3. 选择 **"清空缓存并硬性重新加载"** (Empty Cache and Hard Reload)

### 方法3：清除浏览器缓存 🗑️

#### Chrome/Edge
1. 按 `Ctrl + Shift + Delete` (Mac: `Cmd + Shift + Delete`)
2. 选择时间范围：**最近 1 小时**
3. 勾选：
   - ✅ 浏览历史记录
   - ✅ 缓存的图片和文件
4. 点击 **"清除数据"**

#### Firefox
1. 按 `Ctrl + Shift + Delete`
2. 时间范围：**最近 1 小时**
3. 勾选：
   - ✅ 浏览和下载历史
   - ✅ 缓存
4. 点击 **"立即清除"**

### 方法4：重启 Vite 开发服务器 🔥

如果以上方法都不行，重启前端服务：

```bash
# 在终端中按 Ctrl+C 停止服务

# 然后重新启动
npm run dev
```

或者强制清除 Vite 缓存：

```bash
# 删除 Vite 缓存
rm -rf node_modules/.vite

# 或者 Windows PowerShell:
Remove-Item -Recurse -Force node_modules\.vite

# 然后重新启动
npm run dev
```

## 🔍 调试方法

### 检查 CSS 是否加载

1. 打开浏览器开发者工具（F12）
2. 切换到 **Elements** 标签
3. 找到 `.step-actions` 元素
4. 查看 **Styles** 面板

**应该看到：**
```css
.step-actions {
  display: flex;
  justify-content: center;
  gap: 16px;
  padding: 32px 40px;
  border-top: 2px solid #e4e7ed;
  background: #f5f7fa;
  margin-top: auto;
}
```

### 检查元素结构

在 Elements 面板中，展开 DOM 树：

```html
<div class="step-content">
  <div class="content-body centered">
    <div class="el-alert select-template-alert">
      <!-- Alert 内容 -->
    </div>
  </div>
  <div class="step-actions">
    <button>下一步：填写数据</button>
  </div>
</div>
```

**检查点：**
- ✅ `.content-body` 有 `centered` class
- ✅ `.step-actions` 独立在外层
- ✅ `.step-content` 使用 `display: flex`

## 💡 常见问题

### Q1: 样式还是混在一起
**A:** 
1. 确认使用了强制刷新（Ctrl+Shift+R）
2. 检查浏览器控制台是否有 CSS 错误
3. 尝试隐私/无痕模式打开页面

### Q2: Alert 没有居中
**A:**
1. 检查 `.content-body` 是否有 `centered` class
2. 在 Console 中运行：
```javascript
document.querySelector('.content-body').classList
// 应该包含 'centered'
```

### Q3: 背景色没有显示
**A:**
1. 检查 CSS 是否被其他样式覆盖
2. 在 Styles 面板查看是否有更高优先级的样式
3. 尝试添加 `!important` 进行调试

### Q4: 按钮区域没有分隔
**A:**
1. 检查 `.step-actions` 的 `border-top` 是否生效
2. 确认 `background: #f5f7fa` 已应用
3. 查看 `margin-top: auto` 是否生效

## 🆘 最后手段

如果以上方法都不行：

### 1. 完全重置浏览器缓存
```bash
# Chrome
chrome://settings/clearBrowserData

# Edge
edge://settings/clearBrowserData

# Firefox
about:preferences#privacy
```

### 2. 使用其他浏览器测试
- Chrome
- Edge
- Firefox
- Safari

### 3. 检查文件是否保存

- 查看 Git 状态：`git status`
- 查看最近修改：`git diff`

```bash

# 2. 停止所有 Node 进程
# 3. 重新打开 VSCode
# 4. 重新启动服务
npm run dev
```
